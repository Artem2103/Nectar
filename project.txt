# Nectar

Version: MVP v1

---

# Vision

Nectar is an AI-powered nutrition and health tracking application that makes calorie and macro tracking effortless.

Instead of manually entering meals, users take a photo of their food and receive an AI analysis including calories, protein, carbohydrates, fat, fiber, sugar, sodium, and a health score.

The experience should feel modern, minimal, fast, and premium.

Primary inspiration:
- Cal AI
- Apple Health
- MyFitnessPal (simplicity only)

---

# MVP Goal

Build an Android application using Flutter.

The first version focuses on:

- Beautiful UI
- Excellent UX
- AI meal logging
- Daily nutrition tracking

Backend, authentication, subscriptions and advanced social features can be implemented later.

# Design Philosophy

The application should feel:

- Minimal
- Premium
- Modern
- Clean
- Fast

Avoid clutter.

Lots of whitespace.

Large rounded cards.

Smooth animations.

Apple-quality polish.

---

# Design Rules

Never hardcode:

- colors
- spacing
- font sizes
- border radius
- shadows

Always use theme constants.

Create reusable widgets.

If something appears twice, create a reusable component.

# Coding Rules

Prefer readability over cleverness.

Prefer reusable widgets over duplicated code.

Keep widgets small.

Keep business logic separate from UI.

Follow Flutter best practices.

---

# Workflow

Never implement multiple major features at once.

Always:

1. Explain the plan.
2. Implement.
3. Wait for confirmation.

Do not refactor unrelated code.

Do not change existing UI unless requested.
