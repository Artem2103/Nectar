## Phase 1 — Data Layer  (foundation; everything else depends on this)

- [ ] Add packages: shared_preferences, image_picker, http, uuid
- [ ] MealEntry model  (id, timestamp, name, imagePath, kcal, protein, carbs, fat, fiber, sugar, sodium)
- [ ] WeightEntry model  (id, timestamp, valueKg)
- [ ] UserGoals model  (dailyKcal, proteinG, carbsG, fatG, startWeightKg, goalWeightKg)
- [ ] MealRepository  — CRUD + persist list as JSON in SharedPreferences
- [ ] WeightRepository — CRUD + persist list as JSON in SharedPreferences
- [ ] GoalsRepository  — read/write UserGoals as JSON in SharedPreferences
- [ ] Riverpod providers for each repository (AsyncNotifierProvider)
- [ ] Replace dailySummaryProvider: read today's meals from MealRepository, compute remaining macros vs UserGoals
- [ ] Replace progressSummaryProvider: read WeightRepository + MealRepository (streak logic)

## Phase 2 — Add Meal Flow

- [ ] Wire FAB → /add-meal route (GoRouter push, not shell branch)
- [ ] AddMealScreen: manual form (meal name, kcal, protein, carbs, fat)
- [ ] "Analyse photo" button: image_picker → send image to Claude API → auto-fill form fields

- [ ] On save: pop back to Home; provider invalidation triggers reactive rebuild
